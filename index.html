What is a Shader?

Shaders are scripts that run on the GPU instead of the CPU. Because the GPU works very differently than the CPU, shaders have their own programming languages. Two of the most common shader languages are openGL Shader Language (GLSL), and High-Level Shader Language (HLSL). A JavaScript library called WebGL allows shaders written in GLSL to be used in web applications.

The primary difference between writing code for the GPU and the CPU is that the GPU uses parallel processing. This fundamentally changes the way that code must be written as the same code must be run for every pixel in the shader simultaneously. Pixels cannot see what the other pixels are doing and different pixels cannot take different paths through the code without significant performance penalties. With these limitations comes the power of parallel processing. Code can be run for every pixel on the screen while maintaining high performance, allowing us to render dynamic, complex visuals with minimal performance cost.

Vert and Frag Shaders

Every shader effect is made up of two shaders. The vertex shader, or vert shader, changes the vertices for the shape. 2D shaders, like the ones on this website typically use a very simple vert shader that merely passes information through it without making any changes. The fragment shader, or frag shader, changes every pixel on the shape it is applied to. This is where the complex effects are created using GLSL.
